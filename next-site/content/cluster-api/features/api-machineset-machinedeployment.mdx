## MachineSet

MachineSet 是 Kubernetes **ReplicaSet** 的類比，負責維護一組 Machine 的期望數量。當 Machine 意外消失時，MachineSet controller 會自動補充新的 Machine；當 Machine 數量超過期望值時，controller 會選擇刪除多餘的 Machine。

### MachineSet Spec 欄位

| 欄位 | 類型 | 說明 |
|------|------|------|
| `clusterName` | string | 所屬叢集名稱（不可變）|
| `replicas` | int32 | 目標 Machine 數量 |
| `selector` | LabelSelector | 用於辨識所管轄的 Machine |
| `template.spec.clusterName` | string | 新建 Machine 的叢集名稱 |
| `template.spec.bootstrap.configRef` | ObjectRef | Bootstrap provider 引用 |
| `template.spec.infrastructureRef` | ObjectRef | Infrastructure provider 引用 |
| `template.spec.version` | string | Kubernetes 版本（如 v1.28.0）|
| `deletePolicy` | string | 刪除策略：`Random`、`Newest`、`Oldest`（預設 Random）|
| `minReadySeconds` | int32 | 機器視為 ready 前的最少就緒秒數 |

### MachineSet Status 欄位

| 欄位 | 類型 | 說明 |
|------|------|------|
| `replicas` | int32 | 目前管轄的 Machine 數量 |
| `readyReplicas` | int32 | 就緒的 Machine 數量 |
| `availableReplicas` | int32 | 可用的 Machine 數量 |
| `fullyLabeledReplicas` | int32 | 完全符合 selector label 的 Machine 數量 |
| `observedGeneration` | int64 | 最後觀察到的 generation |
| `conditions` | []Condition | 健康狀態條件 |

### MachineSet YAML 範例

```yaml
apiVersion: cluster.x-k8s.io/v1beta1
kind: MachineSet
metadata:
  name: my-cluster-workers
  namespace: default
spec:
  clusterName: my-cluster
  replicas: 3
  selector:
    matchLabels:
      cluster.x-k8s.io/cluster-name: my-cluster
      cluster.x-k8s.io/set-name: my-cluster-workers
  template:
    metadata:
      labels:
        cluster.x-k8s.io/cluster-name: my-cluster
        cluster.x-k8s.io/set-name: my-cluster-workers
    spec:
      clusterName: my-cluster
      version: v1.28.0
      bootstrap:
        configRef:
          apiVersion: bootstrap.cluster.x-k8s.io/v1beta1
          kind: KubeadmConfigTemplate
          name: my-cluster-workers
      infrastructureRef:
        apiVersion: infrastructure.cluster.x-k8s.io/v1beta1
        kind: AWSMachineTemplate
        name: my-cluster-workers
```

### deletePolicy 說明


<Callout type="tip" title="deletePolicy 三種策略">

- **`Random`（預設）**：隨機選擇要刪除的 Machine，適合一般縮減場景。
- **`Newest`**：優先刪除最新建立的 Machine，有利於快速回滾（保留較穩定的舊機器）。
- **`Oldest`**：優先刪除最舊的 Machine，有利於輪替更新（逐步替換老舊節點）。

</Callout>

## MachineDeployment

MachineDeployment 是 Kubernetes **Deployment** 的類比，透過管理多個 MachineSet 來提供**滾動更新**能力。使用者只需更新 MachineDeployment 的 template，controller 便會自動建立新的 MachineSet 並逐步遷移 Machine，無需手動介入。

### MachineDeployment Spec 欄位

| 欄位 | 類型 | 說明 |
|------|------|------|
| `clusterName` | string | 所屬叢集名稱（不可變）|
| `replicas` | int32 | 目標 Machine 數量 |
| `selector` | LabelSelector | 用於辨識所管轄的 MachineSet |
| `template` | MachineTemplateSpec | Machine 模板規格 |
| `strategy.type` | string | 更新策略：`RollingUpdate` 或 `OnDelete` |
| `strategy.rollingUpdate.maxUnavailable` | IntOrString | 更新時最多不可用數量 |
| `strategy.rollingUpdate.maxSurge` | IntOrString | 更新時最多額外建立數量 |
| `minReadySeconds` | int32 | 最少就緒秒數 |
| `revisionHistoryLimit` | int32 | 保留的歷史 MachineSet 數量（預設 1）|
| `paused` | bool | 暫停 rollout |

### MachineDeployment Status 欄位

| 欄位 | 類型 | 說明 |
|------|------|------|
| `observedGeneration` | int64 | 最後觀察到的 generation |
| `selector` | string | selector 字串 |
| `replicas` | int32 | 總 Machine 數量 |
| `updatedReplicas` | int32 | 已更新到最新版本的數量 |
| `readyReplicas` | int32 | 就緒數量 |
| `availableReplicas` | int32 | 可用數量 |
| `unavailableReplicas` | int32 | 不可用數量 |
| `phase` | string | ScalingUp / ScalingDown / Running / Failed / Unknown |
| `conditions` | []Condition | 健康狀態條件 |

### MachineDeployment YAML 範例（RollingUpdate）

```yaml
apiVersion: cluster.x-k8s.io/v1beta1
kind: MachineDeployment
metadata:
  name: my-cluster-workers
  namespace: default
spec:
  clusterName: my-cluster
  replicas: 5
  selector:
    matchLabels:
      cluster.x-k8s.io/cluster-name: my-cluster
      cluster.x-k8s.io/deployment-name: my-cluster-workers
  template:
    metadata:
      labels:
        cluster.x-k8s.io/cluster-name: my-cluster
        cluster.x-k8s.io/deployment-name: my-cluster-workers
    spec:
      clusterName: my-cluster
      version: v1.28.0
      bootstrap:
        configRef:
          apiVersion: bootstrap.cluster.x-k8s.io/v1beta1
          kind: KubeadmConfigTemplate
          name: my-cluster-workers-v1
      infrastructureRef:
        apiVersion: infrastructure.cluster.x-k8s.io/v1beta1
        kind: AWSMachineTemplate
        name: my-cluster-workers-v1
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxUnavailable: 0    # 不允許不可用（zero-downtime）
      maxSurge: 1          # 每次新建 1 台
```

## RollingUpdate 流程

### Step 1 — 更新觸發

修改 `MachineDeployment.Spec.Template`（例如更改 `infrastructureRef` 指向新版本的 MachineTemplate，或更改 `version` 欄位）。Controller 偵測到 template hash 變更後，開始執行滾動更新流程。

### Step 2 — 建立新 MachineSet

Controller 建立帶有**新 template hash label** 的新 MachineSet；舊 MachineSet 繼續存在，replicas 暫時保持不變。

### Step 3 — 逐步擴展新、縮減舊

1. 根據 `maxSurge`：先擴展新 MachineSet（建立新 Machine）
2. 等待新 Machine 就緒後，根據 `maxUnavailable`：縮減舊 MachineSet（刪除舊 Machine）
3. 重複上述步驟，直到新 MachineSet replicas = 目標數量，舊 MachineSet replicas = 0

### Step 4 — 更新完成

- 新 MachineSet 接管全部 Machine
- 舊 MachineSet 保留（replicas = 0），供回滾使用
- 超過 `revisionHistoryLimit` 的舊 MachineSet 會被自動刪除


<Callout type="tip" title="回滾（Rollback）">

若更新過程出現問題，將 `MachineDeployment.Spec.Template` 改回舊版本，controller 會自動識別對應的舊 MachineSet 並擴展它，同時縮減新 MachineSet，完成回滾操作。

</Callout>

## MachineSet vs MachineDeployment vs MachinePool

| 比較項目 | MachineSet | MachineDeployment | MachinePool |
|---------|-----------|-------------------|-------------|
| 類比 K8s 物件 | ReplicaSet | Deployment | —（雲端節點組）|
| 管理機制 | 直接管理 Machine | 透過 MachineSet | 委託 InfraMachinePool |
| 滾動更新 | ❌ 需手動 | ✅ 自動 | 依 InfraPool 實作 |
| 使用場景 | 靜態 worker 組 | 生產 worker 組 | 雲端 Auto Scaling Group |
| Feature Gate | — | — | MachinePool（Beta，預設開啟）|

## Machine 擁有者鏈


<Callout type="info" title="ownerReference 串接關係">

```
MachineDeployment → MachineSet → Machine → Node
```

每個物件都透過 `ownerReference` 串接。刪除上層物件時，Kubernetes 的 Garbage Collection 機制會自動 cascade 刪除下層物件。

若要保留某台 Machine 不被 MachineSet 刪除（例如需要手動維護），需移除該 Machine 上的 `cluster.x-k8s.io/set-name` label，使其脫離 MachineSet 的管轄範圍。

</Callout>
