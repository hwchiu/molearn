## 🌐 整體觀念：MAAS Provider 解決什麼問題？

![MAAS 生態系定位](/diagrams/maas/overview-position.png)

![MAAS 互動架構](/diagrams/maas/interaction-architecture.png)

MAAS Provider 以標準 CAPI Infrastructure Provider 模式運作。MAAS（Metal as a Service）是 Canonical 開發的裸機雲端管理平台，能夠像管理雲端虛擬機一樣管理實體伺服器。CAPM 的職責是當 CAPI 說「給我一台機器」時，透過 MAAS API 自動分配、部署 OS、設定網路，最終回傳一台可以加入 Kubernetes cluster的實體node。

![架構元件圖](/diagrams/maas/architecture-components.png)

## 🔄 核心流程

平台工程師執行 `kubectl apply` 建立 Cluster 物件後，MAAS Provider 的流程如下：

1. **MaasClusterReconciler** 呼叫 MAAS DNS API，建立 FQDN（`<cluster-name>.<dnsDomain>`），設定 `MaasCluster.Spec.ControlPlaneEndpoint`
2. **MaasMachineReconciler（CP）** 呼叫 MAAS allocate API 選取機器，再呼叫 deploy API 部署 OS + 注入 cloud-init userdata，等待機器上線
3. **MaasClusterReconciler** 偵測到 API Server 上線後，設定 `MaasCluster.Status.Ready = true`
4. **MaasMachineReconciler（Workers）** 為每台 Worker 重複 allocate → deploy 流程，設定 `Node.Spec.ProviderID` 與 `MaasMachine.Status.Ready = true`
5. 刪除cluster時，呼叫 MAAS release API 歸還機器，清除 DNS 記錄

## 🧩 元件總覽

| CRD / Controller | 用途 |
|-----------------|------|
| `MaasCluster` | cluster層級：DNS domain、controlPlaneEndpoint、failure domains |
| `MaasMachine` | 機器層級：分配條件（CPU/記憶體/資源池）、OS image、部署參數 |
| `MaasMachineTemplate` | 不可變模板，供 MachineDeployment 批次建立機器 |
| `MaasClusterReconciler` | 管理 DNS FQDN、監控 API Server 可用性、設定cluster Ready 狀態 |
| `MaasMachineReconciler` | Allocate → Deploy → ProviderID → Ready 完整機器生命週期 |
| `spectrocloud/maas-client-go` | 封裝 MAAS REST API 的 Go SDK |

### 原始碼：MaasCluster 型別

```go
// 原始碼: api/v1beta1/maascluster_types.go

type MaasClusterSpec struct {
	DNSDomain            string      `json:"dnsDomain"`            // MAAS DNS 網域（例如 maas）
	ControlPlaneEndpoint APIEndpoint `json:"controlPlaneEndpoint"` // API Server FQDN:port
	FailureDomains       []string    `json:"failureDomains,omitempty"` // 可用 MAAS Zone 清單
}

type MaasClusterStatus struct {
	Ready          bool                   `json:"ready"`
	Network        Network                `json:"network,omitempty"`  // 含 DNSName
	FailureDomains clusterv1.FailureDomains `json:"failureDomains,omitempty"`
	Conditions     clusterv1.Conditions   `json:"conditions,omitempty"`
}

// APIEndpoint 代表 kube-apiserver 對外的端點
type APIEndpoint struct {
	Host string `json:"host"` // FQDN（由 DNS service 建立）
	Port int    `json:"port"`
}
```

### 原始碼：Reconcile 進入點

```go
// 原始碼: controllers/maascluster_controller.go

// MaasClusterReconciler reconciles a MaasCluster object
type MaasClusterReconciler struct {
	client.Client
	Log                 logr.Logger
	Scheme              *runtime.Scheme
	Recorder            record.EventRecorder
	GenericEventChannel chan event.GenericEvent // API Server 上線通知
	Tracker             *remote.ClusterCacheTracker
}

func (r *MaasClusterReconciler) Reconcile(ctx context.Context, req ctrl.Request) (_ ctrl.Result, rerr error) {
	maasCluster := &infrav1beta1.MaasCluster{}
	r.Client.Get(ctx, req.NamespacedName, maasCluster)

	// 建立 clusterScope（封裝 DNS 操作、API Server 探活邏輯）
	clusterScope, _ := scope.NewClusterScope(scope.ClusterScopeParams{ ... })
	defer clusterScope.Close()

	// 將 spec.failureDomains 轉換為 CAPI 標準 FailureDomains map
	failureDomains := make(clusterv1.FailureDomains)
	for _, az := range maasCluster.Spec.FailureDomains {
		failureDomains[az] = clusterv1.FailureDomainSpec{ControlPlane: true}
	}
	maasCluster.Status.FailureDomains = failureDomains

	// 依 DeletionTimestamp 分流
	if !maasCluster.DeletionTimestamp.IsZero() {
		return r.reconcileDelete(ctx, clusterScope)
	}
	return r.reconcileNormal(ctx, clusterScope)
}

func (r *MaasClusterReconciler) reconcileNormal(_ context.Context, clusterScope *scope.ClusterScope) (ctrl.Result, error) {
	// 呼叫 MAAS DNS API 建立 FQDN
	dnsService.ReconcileDNS()

	// 設定 ControlPlaneEndpoint 並標記cluster Ready
	maasCluster.Spec.ControlPlaneEndpoint = infrav1beta1.APIEndpoint{
		Host: maasCluster.Status.Network.DNSName,
		Port: clusterScope.APIServerPort(),
	}
	maasCluster.Status.Ready = true

	// 持續監控 API Server 可用性
	clusterScope.ReconcileMaasClusterWhenAPIServerIsOnline()
	return ctrl.Result{}, nil
}
```

## ⚠️ 常見陷阱

<Callout type="warning" title="使用前須知">

- **固定 Pod CIDR**：Pod 網路使用 `192.168.0.0/16`，目前無法透過 `MaasCluster` spec 自訂
- **不支援原地升級**：Kubernetes 版本升級需透過 rolling MachineDeployment（刪舊建新）
- **MAAS 版本需求**：需 MAAS ≥ 3.5.10，目標機器至少 16GB RAM
- **單一端點限制**：一個 Provider 實例僅對應一個 MAAS 端點
- MAAS 認證透過環境變數 `MAAS_ENDPOINT` 與 `MAAS_API_KEY` 設定（非 kubeconfig）

</Callout>
